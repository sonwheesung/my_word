$ARGUMENTS 화면(파일 또는 "all")에 대해 다양한 디바이스 해상도에서 UI가 정상 표시되는지 테스트해줘.

Puppeteer MCP를 사용하여 Expo 웹 앱(http://localhost:8081)에 접속하고, 아래 디바이스별 뷰포트 크기로 스크린샷을 찍어 레이아웃 문제를 확인해.

## 테스트 절차

1. `puppeteer_launch`로 브라우저 실행
2. 각 디바이스별로:
   a. `puppeteer_evaluate`로 뷰포트 크기 변경: `await page.setViewport({ width: W, height: H })`
   b. 대상 화면으로 네비게이션
   c. `puppeteer_screenshot`으로 스크린샷 캡처
   d. 아래 체크리스트 검증

## 체크리스트

### 1. 레이아웃
- 콘텐츠가 화면 밖으로 잘리지 않는지
- 가로 스크롤이 불필요하게 생기지 않는지
- 버튼이 화면 하단에 가려지지 않는지
- 요소 간 겹침이 없는지

### 2. 텍스트
- 텍스트가 잘리거나 말줄임표(...)로 표시되지 않는지
- 폰트 크기가 너무 작아 읽기 어렵지 않는지 (최소 12px)
- 긴 텍스트가 줄바꿈 되어 레이아웃을 깨뜨리지 않는지

### 3. 터치 영역
- 버튼/터치 영역이 최소 44x44 이상인지
- 요소 간 간격이 너무 좁아 오터치 가능성이 없는지

### 4. 스크롤
- 스크롤이 필요한 콘텐츠가 ScrollView로 감싸져 있는지
- 고정 요소(하단 버튼, 배너 등)가 스크롤에 영향받지 않는지

### 5. 폴더블/태블릿 대응
- 넓은 화면에서 콘텐츠가 과도하게 늘어나지 않는지
- maxWidth 제한이 있는지
- 좁은 화면(Z Fold 접힌)에서 요소가 겹치지 않는지

### 6. 광고 배너
- 배너 광고가 콘텐츠를 가리지 않는지
- 배너 영역만큼 콘텐츠 영역이 줄어들었는지

## 결과 보고 형식

각 디바이스별로 아래 형식으로 보고:

```
## [디바이스명] (너비 x 높이)
- 상태: PASS / WARN / FAIL
- 스크린샷: (캡처 완료)
- 발견 사항:
  - [심각도] 설명
```

심각도: CRITICAL (사용 불가) / MEDIUM (UX 저하) / LOW (개선 권장)

문제 발견 시 소스 코드를 읽고 수정 방안을 제안해. 수정 후 `npx tsc --noEmit`으로 검증해.
