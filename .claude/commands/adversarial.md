$ARGUMENTS 화면(파일)에 대해 악의적/비정상 입력 테스트를 수행해줘.

대상 파일을 읽고 아래 체크리스트를 **각 항목당 최소 10개 시나리오**로 검증해.

## 체크리스트

### 1. 입력 필드 (TextInput)
- maxLength 설정 여부
- trim() 처리 여부 (저장, 비교, 전송 모든 곳)
- 빈값/공백만 입력 시 처리
- 특수문자 입력 (< > " ' & \ / 등)
- 이모지 입력
- 초장문 입력 (maxLength 경계값)
- 줄바꿈 문자 입력 (\n \r)

### 2. 버튼/액션
- 더블탭 방지 (loading/isSubmitting 가드)
- 비동기 작업 중 UI 비활성화
- 빈 상태에서 저장/제출 시도

### 3. 배열/리스트
- 무한 추가 방지 (개수 제한)
- 빈 배열 접근 (arr[0] 전 length 체크)
- 모든 항목 삭제 시도 (최소 1개 유지 여부)

### 4. 계산/표시
- 0 나누기 가드
- undefined/null 접근 시 ?. 또는 ?? 사용
- 숫자 오버플로우

### 5. 네트워크/비동기
- async 함수에 try-catch 존재 여부
- 에러 시 사용자 피드백 (Toast/Alert)
- 로딩 상태 표시 여부

### 6. 레이아웃
- 내용이 많을 때 버튼이 가려지지 않는지
- ScrollView 내 고정 요소 처리

## 결과 보고 형식

발견된 버그마다 아래 형식으로 보고:

```
[심각도] BUG: 설명
- 파일: 파일명:라인번호
- 재현: 시나리오 설명
- 수정: 제안
```

심각도: CRITICAL (기능 장애) / MEDIUM (UX 문제) / LOW (개선 권장)

버그 발견 시 수정까지 진행하고 `npx tsc --noEmit`으로 검증해.
